## 2024-05-24 - Throttling High-Frequency Layout Reads in React
**Learning:** High-frequency events (like `mousemove`) that read synchronous layout properties (`getBoundingClientRect()`) cause severe layout thrashing and frame drops. When optimizing this with `requestAnimationFrame` inside a React hook (like `useMouseGlow`), storing the animation frame ID directly in a variable inside the callback can lead to stale closures or lost frame IDs across renders.
**Action:** Always use `useRef` to store the animation frame ID to correctly persist state across renders without causing re-renders, and ensure the pending frame is cancelled on component unmount to prevent memory leaks and state updates on unmounted components. Also, mark the event listener with `{ passive: true }` when possible.
